# ASTRONOMICON ADVISORY BUFFER V0.1

- owner: Astronomicon
- status: RAW_ADVISORY_AND_IDEA_BUFFER
- purpose: preserve external ideas, plans, audits, architectural proposals, examples, and research outputs before they become formal TASK_ID records.

## Why this exists

Not every useful idea should immediately become a TASK_ID.

Some materials are:
- Kiro/Speculum/Logos advisory reports;
- rough Owner ideas;
- visual/design research;
- architecture proposals;
- examples of good or bad implementation;
- planning impulses;
- external analysis that must not be lost.

These inputs must be preserved and later reviewed, filtered, and converted into formal General Tasks / Tasks / Stages only when appropriate.

## Rules

1. Advisory buffer is not execution authority.
2. Raw advisory files do not automatically become canon.
3. Logos-Prime, Speculum, Servitors, and future organs may read this zone for context.
4. Useful ideas must be explicitly promoted into ASTRONOMICON task structures before execution.
5. Bad or rejected ideas should be preserved with verdicts when useful for preventing repeated mistakes.
6. This zone is especially important before Arc 5, where IMPERIUM will test whether it can help write/build itself.
