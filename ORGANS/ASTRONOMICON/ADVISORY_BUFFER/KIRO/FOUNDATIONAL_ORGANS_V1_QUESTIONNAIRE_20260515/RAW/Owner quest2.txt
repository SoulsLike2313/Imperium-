# 9. Red-Team Notes

## Risks If Questions Are Skipped

| Skipped Question Category | Risk | Severity |
|---------------------------|------|----------|
| Atomic unit definitions (ASTRO-01, ADMIN-01, etc.) | Organs build incompatible data models | CRITICAL |
| State ownership (ASTRO-02, ADMIN-02) | Two organs fight over state machine, causing conflicts | CRITICAL |
| Evidence format (CROSS-01) | Organs produce incompatible evidence, breaking cross-organ checks | HIGH |
| No-fake-green proof (CROSS-07) | Dashboards show green without real evidence, masking failures | CRITICAL |
| Acceptance authority (DOCTR-04) | Advisory material treated as canonical, causing doctrine drift | HIGH |
| Boundary definitions (all D sections) | Organs steal each other's functions, creating tangled dependencies | HIGH |

## Risks If Questions Are Answered Vaguely

| Vague Answer Pattern | Risk | Example |
|----------------------|------|---------|
| "depends" / "maybe" / "sometimes" | No clear implementation path | "ASTRO-02: depends on context" |
| "all of the above" without priority | Feature creep, no MVP | "ADMIN-03: all fields mandatory" |
| "owner decides later" for blockers | Hardening stalls waiting for decisions | "DOCTR-09: owner decides" |
| "flexible" / "configurable" | No default behavior, endless options | "CROSS-08: flexible per user" |

## Specific Red Flags

1. **If ASTRO-02 and ADMIN-02 both say "owns_exclusively"** → Conflict. One must yield.

2. **If CROSS-07 does not include "evidence_file_validates_against_schema"** → Fake green risk. Empty or malformed evidence could pass.

3. **If DOCTR-04 is not "owner_only"** → Doctrine could be canonized without Owner awareness.

4. **If OFFAG-03 is "auto_assign" but ADMIN-07 is "owner_only"** → Agents can be assigned but cannot complete tasks. Deadlock.

5. **If CROSS-10 is "per_organ_choice"** → No unified Sanctum. Each organ becomes isolated silo.

6. **If any dashboard question allows "display_only" but corresponding role question allows "enforces"** → Dashboard cannot trigger enforcement. Mismatch.

## Pre-Hardening Checklist

Before starting hardening, verify:

- [ ] All CRITICAL blocker questions have clear answers
- [ ] No contradictory answers between organs
- [ ] No contradictory answers between role and dashboard for same organ
- [ ] CROSS-07 (no-fake-green) has at least 3 proof requirements
- [ ] All "owner_only" answers are acceptable to Owner (not too restrictive)
- [ ] All "auto" answers are acceptable to Owner (not too permissive)
- [ ] Deferred items are explicitly listed with rationale
