# Doctrinarium MVP Advisory

This is a substantial architecture advisory request, not a standard spec workflow. You're asking for a detailed design consultation before any implementation begins.

Let me provide the comprehensive advisory you've requested.

---

## A. Best Conceptual Form

**Doctrinarium should be the law enforcement and organ health gate of IMPERIUM.**

### What Doctrinarium Owns

| Ownership | Description |
|-----------|-------------|
| Law Registry | Canonical store of all active, retired, and draft laws |
| Doctrine Registry | Operational doctrines derived from laws |
| Law Hierarchy | Higher/lower law relationships and conflict resolution |
| Law Compliance Checks | Scripts that evaluate whether actions/states violate laws |
| Organ Form Gates | Checkers that evaluate whether organs meet minimum working form |
| Task-Start Permission Verdict | Binary gate: can execution proceed or not |
| Law Integrity Guard | Protection against law corruption, duplication, contradiction |
| Violation Records | Evidence of law violations (pending Inquisition) |
| Inquisition Hook (disabled) | Prepared escalation path, not active |

### What Doctrinarium Must NOT Own

| Not Owned | Belongs To |
|-----------|------------|
| Task decomposition, stage maps, corridor flow | Astronomicon |
| Address book, chronicle, memory, bundles | Administratum |
| Role contracts, modes, agent behavior | Officio Agentis |
| Tool implementation, script machinery | Mechanicus |
| Audit verdicts, investigation, punishment | Inquisition |
| UI/dashboard rendering | Sanctum |
| Execution itself | Servitor |

### Authority Model

Doctrinarium has **advisory authority with blocking power**:
- It does not execute tasks
- It does not assign roles
- It does not manage lifecycle
- It **can block** task execution by refusing to issue ALLOW_EXECUTION verdict
- It **can warn** without blocking
- It **cannot override Owner** — Owner can force-proceed with recorded exception

### What "Law Active" Means

A law is active when:
1. `status: "active"` in law registry
2. `effective_date` ≤ current date
3. `expiry_date` is null or > current date
4. No `superseded_by` pointing to another active law
5. Law passes integrity check (valid schema, no contradictions)
6. Law has at least one enforcement mode defined

### What "Organ Legally/Operationally Alive" Means

An organ is alive when:
1. Folder exists at `ORGANS/{ORGAN_NAME}/`
2. `README.md` exists and is non-empty
3. Registry entry exists in `ORGAN_REGISTRY.json`
4. Self-report file exists and is fresh (< 24h for active organs, < 7d for deferred)
5. No BLOCKER-level violations recorded
6. Required checker script exists and last ran with PASS or PASS_WITH_WARNINGS
7. Organ does not claim ownership of another organ's domain

### What "Task Execution Allowed" Means

Task execution is allowed when:
1. All required organs are alive (Astronomicon, Administratum, Officio Agentis, Doctrinarium)
2. No BLOCKER-level law violations exist for the task scope
3. Assigned agent role is valid per Officio Agentis
4. Task is registered in Astronomicon
5. No unresolved exceptions requiring Owner approval
6. Law integrity check passes
7. Organ health check passes with no BLOCKERs

## C. Higher and Lower Laws

### Law Hierarchy

| Level | Name | Authority | Override | Example |
|-------|------|-----------|----------|---------|
| 0 | **Throne Laws** | Owner only | Cannot be overridden | "No commit from VM2" |
| 1 | **Constitutional Laws** | Owner + Strategium | Only by Throne | "All artifacts must have provenance" |
| 2 | **Organ Laws** | Organ owner | Only by Constitutional+ | "Administratum owns chronicle" |
| 3 | **Operational Doctrines** | Task scope | Only by Organ+ | "Bundle must include hash manifest" |
| 4 | **Task Rules** | Single task | Expire with task | "This task uses dry-run mode" |

### Conflict Resolution

1. Higher law always wins over lower law
2. If same level conflicts: newer law wins unless explicitly marked `conflict_resolution: "older_wins"`
3. Exceptions require Owner approval recorded in `exception_approval.json`
4. Lower law **cannot** override higher law — this is a BLOCKER violation

### Law Retirement

- Laws are never deleted, only marked `status: "retired"` or `status: "superseded"`
- Retired laws remain in registry with `retired_utc` timestamp
- Superseded laws point to replacement via `superseded_by` field
- Stale law = active law not referenced by any gate/check in 90 days → WARNING

### Exception Approval

```json
{
  "exception_id": "EXC-20260515-001",
  "law_id": "LAW-001",
  "task_id": "TASK-20260515-...",
  "reason": "One-time migration requires temporary bypass",
  "approved_by": "Owner",
  "approved_utc": "2026-05-15T10:00:00Z",
  "expires_utc": "2026-05-16T10:00:00Z",
  "evidence_path": "ARTIFACTS/.../exception_approval.json"
}
```

## D. Machine-Readable Law Format

**Recommendation: JSON as canonical, paired with `.md` for human reading.**

### `law.schema.json` — Core Fields

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["law_id", "title", "law_level", "authority", "status", "version", "scope", "applies_to", "enforcement_mode", "created_utc"],
  "properties": {
    "law_id": { "type": "string", "pattern": "^LAW-[0-9]{3,}$" },
    "title": { "type": "string", "maxLength": 120 },
    "law_level": { "type": "integer", "minimum": 0, "maximum": 4 },
    "authority": { "type": "string", "enum": ["Owner", "Strategium", "Organ", "Task"] },
    "status": { "type": "string", "enum": ["draft", "active", "retired", "superseded"] },
    "version": { "type": "string", "pattern": "^[0-9]+\\.[0-9]+$" },
    "scope": { "type": "string", "enum": ["global", "organ", "task", "artifact"] },
    "applies_to": { "type": "array", "items": { "type": "string" } },
    "prohibited_actions": { "type": "array", "items": { "type": "string" } },
    "required_actions": { "type": "array", "items": { "type": "string" } },
    "gate_checks": { "type": "array", "items": { "type": "string" } },
    "metrics": { "type": "array", "items": { "type": "string" } },
    "violation_conditions": { "type": "array", "items": { "type": "string" } },
    "exception_policy": { "type": "string", "enum": ["no_exceptions", "owner_approval", "organ_approval"] },
    "owner_approval_required": { "type": "boolean" },
    "evidence_required": { "type": "array", "items": { "type": "string" } },
    "enforcement_mode": { "type": "string", "enum": ["blocker", "warning", "advisory", "disabled"] },
    "related_laws": { "type": "array", "items": { "type": "string" } },
    "supersedes": { "type": ["string", "null"] },
    "superseded_by": { "type": ["string", "null"] },
    "effective_date": { "type": "string", "format": "date" },
    "expiry_date": { "type": ["string", "null"], "format": "date" },
    "created_utc": { "type": "string", "format": "date-time" },
    "updated_utc": { "type": ["string", "null"], "format": "date-time" },
    "provenance": { "type": "string" },
    "human_summary": { "type": "string" },
    "human_summary_ru": { "type": "string" }
  }
}
```

### Why JSON over YAML

| Factor | JSON | YAML |
|--------|------|------|
| Schema validation | Native jsonschema | Requires conversion |
| Existing IMPERIUM pattern | All registries are JSON | None |
| Parsing reliability | Strict | Whitespace-sensitive |
| Tool support | Universal | Variable |

**Verdict: JSON canonical, `.md` paired for human reading.**

## E. Human-Readable Law Format

### `LAW-001.md` Template

```markdown
# LAW-001: No Commit From VM2

**Level:** 0 (Throne Law)  
**Authority:** Owner  
**Status:** Active  
**Version:** 1.0  
**Effective:** 2026-05-01  

## Summary

VM2 is a read-only workspace. All commits must originate from PC.

## Scope

- Applies to: all agents, all tasks, all organs
- Enforcement: BLOCKER

## Prohibited Actions

1. `git commit` executed on VM2
2. `git push` executed on VM2
3. Any script that commits from VM2

## Required Actions

1. All patches prepared on VM2 must be bundled for PC transfer
2. PC Owner must review and commit

## Evidence Required

- Git log showing commit origin
- Bundle transfer receipt

## Gate Checks

- `scripts/verify_repo.py` — checks no VM2 commits in history
- `TOOLS/run_administratum_git_cli_check.sh` — validates git state

## Machine Definition

See: `ORGANS/DOCTRINARIUM/LAWS/LAW-001.json`

---

*Комментарий для Owner: Это закон уровня Трона. VM2 никогда не коммитит. Все изменения идут через бандл на PC.*
```

### Law Review Workflow

1. Draft law created with `status: "draft"`
2. Law reviewed by Owner
3. Owner approval recorded in `law_change_receipt.json`
4. Law status changed to `status: "active"`
5. Previous version (if exists) marked `status: "superseded"`
6. Registry updated
7. Integrity check run

## K. Inquisition Hook

### When Doctrinarium Would Call Inquisition

1. BLOCKER-level law violation detected
2. Organ health BLOCKER persists >24h
3. Unapproved law edit detected
4. Fake green pattern detected
5. Ownership boundary violation
6. Evidence tampering suspected

### `inquisition_hook_packet.schema.json`

```json
{
  "type": "object",
  "required": ["packet_id", "trigger_type", "severity", "created_utc", "hook_status"],
  "properties": {
    "packet_id": { "type": "string", "pattern": "^INQ-[0-9]{8}-[0-9]{3}$" },
    "trigger_type": { "type": "string", "enum": ["law_violation", "organ_blocker", "unapproved_edit", "fake_green", "ownership_violation", "evidence_tampering"] },
    "severity": { "type": "string", "enum": ["warning", "blocker", "critical"] },
    "source_organ": { "type": "string" },
    "target_organ": { "type": ["string", "null"] },
    "task_id": { "type": ["string", "null"] },
    "law_id": { "type": ["string", "null"] },
    "description": { "type": "string" },
    "evidence_paths": { "type": "array", "items": { "type": "string" } },
    "created_utc": { "type": "string", "format": "date-time" },
    "hook_status": { "type": "string", "enum": ["disabled", "pending", "sent", "acknowledged"] },
    "disabled_reason": { "type": "string", "default": "Inquisition organ not yet built" }
  }
}
```

### Disabled Hook Behavior

1. Packet is created and saved to `ORGANS/DOCTRINARIUM/INQUISITION_HOOKS/`
2. `hook_status: "disabled"` is set
3. `disabled_reason: "Inquisition organ not yet built"` is recorded
4. Verdict includes `INQUISITION_REQUIRED_BUT_DISABLED`
5. This is NOT fake green — it's honest about the limitation
6. When Inquisition is built, hook status changes to `pending` and packets are processed
