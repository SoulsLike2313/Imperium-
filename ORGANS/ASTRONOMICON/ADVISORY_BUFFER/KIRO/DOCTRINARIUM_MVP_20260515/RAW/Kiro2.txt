### Script Contract: `doctrinarium_task_start_gate_v0_1.py`

```python
"""
Doctrinarium Task Start Gate v0.1

Input:
  --task-id TASK-YYYYMMDD-NNNN-...
  --requesting-agent <agent_name>
  --target-organs <comma-separated list>
  --claimed-role <role_id>
  --claimed-mode <mode_id>

Output:
  - task_start_gate_verdict_{task_id}.json in REPORTS/
  - Exit code 0 = ALLOW_EXECUTION or ALLOW_WITH_WARNINGS
  - Exit code 1 = BLOCK_EXECUTION or STOP_PENDING_OWNER_APPROVAL
  - Exit code 2 = INQUISITION_REQUIRED_BUT_DISABLED

Checks performed:
  1. Law integrity (calls doctrinarium_check_law_integrity_v0_1.py)
  2. Organ health (calls doctrinarium_evaluate_organ_health_v0_1.py)
  3. Role validity (reads OFFICIO_AGENTIS/ROLES/)
  4. Task registration (reads ASTRONOMICON registry)
  5. Law compliance for task scope
  6. Exception status
"""
```

### Fake Green Prevention Tests

```python
# test_fake_green_prevention.py

def test_pass_without_evidence_is_rejected():
    """
    A gate verdict of PASS without evidence_paths is fake green.
    Must be rejected.
    """
    verdict = {
        "task_id": "TASK-TEST-001",
        "verdict": "ALLOW_EXECUTION",
        "evidence_paths": []  # Empty = fake green
    }
    result = validate_verdict(verdict)
    assert result["valid"] == False
    assert "fake_green" in result["errors"]

def test_pass_with_warnings_requires_warning_list():
    """
    PASS_WITH_WARNINGS without actual warnings is fake green.
    """
    verdict = {
        "verdict": "ALLOW_WITH_WARNINGS",
        "warnings": []  # Empty = fake green
    }
    result = validate_verdict(verdict)
    assert result["valid"] == False

def test_organ_healthy_without_checker_run_is_fake_green():
    """
    Organ cannot be HEALTHY if checker never ran.
    """
    health = {
        "organ_name": "TEST_ORGAN",
        "combined_verdict": "HEALTHY",
        "checker_last_run_utc": None  # Never ran
    }
    result = validate_organ_health(health)
    assert result["valid"] == False
    assert "fake_green" in result["errors"]
```

## T. Risks and Red-Team Warnings

### Critical Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| **Ceremonial law with no enforcement** | CRITICAL | Every law MUST have `enforcement_mode` and at least one `gate_check` |
| **Vague laws** | HIGH | `violation_conditions` must be machine-testable, not prose |
| **Law contradiction undetected** | CRITICAL | Integrity checker must cross-reference all laws |
| **Fake green gate** | CRITICAL | Verdict MUST include evidence_paths; empty = invalid |
| **Stale organ health** | HIGH | Freshness thresholds enforced; stale = WARNING/BLOCKER |
| **Unbounded authority** | HIGH | Each law must specify `authority` and `scope` |
| **Doctrinarium stealing roles** | MEDIUM | Ownership boundary checks in integrity checker |
| **Inquisition hook pretending to run** | HIGH | `hook_status: "disabled"` must be explicit; no silent skip |
| **Advisory prompt injection** | MEDIUM | Laws stored as JSON, not executable; prompts separate |
| **Owner approval not recorded** | HIGH | All exceptions require `exception_approval.json` |
| **Language/encoding pollution** | MEDIUM | Schema enforces English for machine fields |
| **Dashboard pretending backend works** | CRITICAL | Dashboard must read actual reports, not mock data |

### Red-Team Scenarios to Test

1. **Create law with `enforcement_mode: "advisory"` and claim compliance** → Must show WARNING, not PASS
2. **Mark organ HEALTHY without running checker** → Must be detected as fake green
3. **Create two laws with same `law_id`** → Must FAIL integrity check
4. **Create lower law that contradicts higher law** → Must BLOCKER
5. **Set `hook_status: "sent"` when Inquisition doesn't exist** → Must be detected as invalid
6. **Approve exception without Owner signature** → Must be rejected
7. **Claim PASS with empty `evidence_paths`** → Must be rejected as fake green
8. **Self-report with future timestamp** → Must be rejected as invalid
