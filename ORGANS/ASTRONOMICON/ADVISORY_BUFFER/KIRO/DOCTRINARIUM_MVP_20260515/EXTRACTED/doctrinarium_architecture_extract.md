# Doctrinarium Architecture Extract (Advisory Reference)

This document is extracted from Kiro advisory input and is not accepted canon.

## Advisory Scope

- Source class: external advisory reference.
- Canon status: not accepted.
- Promotion path: Owner + Logos review and explicit task registration.

## What Doctrinarium Should Own (Advisory)

- Canonical law registry and law format governance.
- Doctrine/law consistency rules.
- Law compliance verification contracts.
- Organ health gate definitions and task start gate definitions.

## What Doctrinarium Must Not Own (Advisory)

- Task lifecycle orchestration (Astronomicon domain).
- Address book/chronicle backend ownership (Administratum domain).
- Role contract ownership (Officio Agentis domain).
- Runtime dashboard ownership (Sanctum domain).

## Authority Model (Advisory)

- Laws are authoritative only after explicit acceptance.
- Registry integrity and provenance are mandatory for accepted laws.
- Advisory text alone has zero canonical authority.

## Meaning of "law active" (Advisory)

- A law is active only when explicitly registered with accepted status and valid provenance.

## Meaning of "organ alive" (Advisory)

- Organ self-report and required health checks are current and pass required freshness/port conditions.

## Meaning of "task execution allowed" (Advisory)

- Task start gate passes against active laws and health constraints.

## Interaction Model (Advisory)

- Astronomicon: consumes gate verdicts, does not delegate law ownership.
- Administratum: stores task evidence and registries, does not own law canon.
- Officio Agentis: supplies role behavior contracts used by law checks.
- Inquisition (future): may receive violation/investigation packets when enabled.
- Sanctum: displays law/health/task gate states once integration is approved.
