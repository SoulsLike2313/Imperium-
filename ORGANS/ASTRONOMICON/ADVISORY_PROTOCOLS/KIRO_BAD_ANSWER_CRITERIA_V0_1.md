# KIRO BAD ANSWER CRITERIA V0.1

## Answer is bad if it contains any of the following

- abstract architecture advice without concrete paths
- recommendations without checker/receipt mapping
- vague verbs (`improve`, `modernize`, `integrate`) without task boundary
- advisory treated as canon
- scaffold treated as implemented
- dashboard proposals before backend truth/stale-status foundation
- commit/push automation proposals before fail-closed receipts
- Owner pain points ignored or softened away
- blockers hidden, minimized, or omitted
- no tables as primary response structure
- no risk classification
- no Owner gate classification
- no exact current context (HEAD, contour status, scope)

## Mandatory consequence

- If any criterion is hit, answer must be marked `NEEDS_REWRITE` or `REJECT_LOW_VALUE_ADVICE`.
