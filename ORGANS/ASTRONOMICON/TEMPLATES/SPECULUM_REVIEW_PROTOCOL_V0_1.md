# SPECULUM REVIEW PROTOCOL V0.1

## Sequence (MVP)

1. Task review first (selected Local Task only).
2. Stage review second (stage map for selected Local Task).

## MVP Scope Rule

- Review all candidates is not required initially.
- Only one selected Local Task proceeds through Speculum review in MVP.

## Authority Rule

- Speculum output is advisory until imported/validated and Owner-approved.
- Speculum output is not canon by itself.

## Structure Rule

- Speculum must fill required structured fields defined by schemas.
- Free-form commentary is limited to `advisory_notes`.
- Missing required fields means review is invalid.

## Gate Rule

- No valid imported task review = no modernization.
- No valid imported stage review = no registration progress.
