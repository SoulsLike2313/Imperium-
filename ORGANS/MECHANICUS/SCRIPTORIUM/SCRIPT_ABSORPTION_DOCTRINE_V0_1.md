# SCRIPT ABSORPTION DOCTRINE V0.1

## Doctrine Intent
Useful scripts produced during work are operational assets of IMPERIUM and must be absorbed into reusable, controlled SCRIPTORIUM practice.

## Core Rules
1. Scripts are assets, not disposable chat fragments.
2. Every useful script must be grouped, described, and made reusable.
3. Script-first does not mean uncontrolled script sprawl.
4. Every script must declare:
- inputs
- outputs
- safe scope
- danger level
- receipts
- known limits
5. Scripts should be Python-first unless another language is clearly justified.
6. Scripts must remain lightweight and operational.

## Required Script Groups
- CHECKERS
- BUILDERS
- SCANNERS
- REGISTRARS
- EXPORTERS
- REPAIR_TOOLS
- VISUAL_TOOLS
- GATE_RUNNERS
- REPO_RECON

## Admission Expectations
- Script includes clear purpose and operator safety notes.
- Script has a deterministic output path or explicit no-output behavior.
- Script has verification signal (receipt/report/log path).
- Script can be reused by future Servitors without hidden context.
