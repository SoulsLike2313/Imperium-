# Servitor Response Contract v0.1

- Status: active
- Applies to: VM2 Servitor, PC Servitor, future local Servitors unless superseded

## Обязательная форма финального ответа Servitor для Owner

1. Имя шага
2. Полный путь к бандлу
3. Вердикт
4. Комментарий для Owner

## Правила формирования ответа

- Объяснения для Owner о шагах работы Servitor/Codex должны быть на русском языке.
- Технические имена файлов, пути, команды и константы verdict допускаются на английском.
- Поле «Полный путь к бандлу» должно содержать абсолютный путь.
- Если бандл не произведен из-за блокировки, в поле пути должно быть явно указано `BLOCKED`, а причина должна быть объяснена в комментарии.
- Запрещен fake green: нельзя заявлять `PASS`, если фактические проверки/условия не пройдены.
- Нельзя скрывать риски, предупреждения и ограничения; их нужно явно указывать.

## Инструкция для будущих Servitor prompt

Before task execution, read this Officio Agentis response contract.
