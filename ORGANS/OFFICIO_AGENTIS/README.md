# Officio Agentis

Officio Agentis defines machine-checkable agent role contracts, role modes, response contracts, and role behavior validation tests for IMPERIUM.

## Purpose

- Build deterministic role contracts for execution, planning, red-team review, and Owner support.
- Provide role-level guardrails for question policy, evidence policy, stop policy, and execution boundaries.
- Keep role behavior testable through JSON contracts and dry-run validators.

## What Officio Agentis Owns

- Role contracts.
- Mode definitions.
- Response contracts.
- Agent settings.
- Question policy.
- Evidence policy.
- Stop policy.
- Prompt writing policy.
- Execution boundary policy.
- Handoff contracts.
- Agent verification tests.

## What Officio Agentis Must Not Own

- Astronomicon task lifecycle and stage-map ownership.
- Administratum address book, chronicle, and task lifecycle backend ownership.
- Doctrinarium legal doctrine ownership.
- Mechanicus toolchain and automation machinery ownership.
- Inquisition final compliance verdict ownership.
- Sanctum UI/dashboard ownership.
- Scriptorium documentation generation ownership.
- Arsenal reusable asset ownership.

## Target Roles

- `SERVITOR`
- `LOGOS_PRIME`
- `LOGOS_SPECULUM`
- `ADVISOR_SERVITOR`

## Canonical Language Rule

- Canonical machine-readable artifacts are English-only by default.
- Russian is used for live Owner-facing comments and controlled UI/i18n contexts.

## No Fake Green

- No PASS without evidence.
- No ownership theft from other organs.
- No unsupported claims of validation.
