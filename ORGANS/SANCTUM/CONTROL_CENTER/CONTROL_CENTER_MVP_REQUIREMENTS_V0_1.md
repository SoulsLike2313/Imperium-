
# CONTROL CENTER MVP REQUIREMENTS V0.1

## Purpose
- Provide Owner-facing operational cockpit for whole IMPERIUM state, progress, control actions, warnings, and delivery readiness.

## Required Panels
- System State.
- Task State.
- Progress percentages.
- Warnings/errors.
- Gates.
- Receipts.
- Diffs/touched paths.
- Memory/context.
- Scripts/tools.
- Capabilities.
- Commands.
- Action cards.
- Agent activity.
- Delivery readiness.
- Owner decision controls.

## Source Binding Requirements
- Every panel value must map to concrete source path/feed contract.
- No PASS/progress claim without gate/receipt evidence.

## Bilingual Requirement
- RU/EN switch is mandatory for Owner-facing text surfaces.

## Must Never Be Fake
- No synthetic green status.
- No performance claim without valid prerequisites.
- No readiness claim while blockers remain unresolved.
