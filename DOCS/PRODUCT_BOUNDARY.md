# PRODUCT BOUNDARY

IMPERIUM v0.1 verification spine defines strict boundaries for what is product skeleton versus legacy/transitional material.

## Boundary Zones

- source: `src/imperium/` contains reusable spine code.
- tools: `TOOLS/` contains operational wrappers and legacy utility scripts.
- scripts: `scripts/` contains callable verification gates and top-level repository verifier.
- schemas: `schemas/` contains JSON schemas and schema registry.
- tests: `tests/` contains minimal repeatable checks for new spine behavior.
- docs: `DOCS/` contains product boundary rules and lifecycle doctrine.
- runtime: `.imperium_runtime/` contains generated runtime reports and receipts.
- artifacts: `ARTIFACTS/` stores historical evidence bundles and transitional outputs.
- archive/local-private: private and transfer-only paths remain outside public repo commitments.

## Transitional Status

This repository is transitional and not product-ready.

Verification Spine v0.1 introduces reproducible structure and gate execution, but legacy folders, historical receipts, and broader organ migration remain unresolved debt pending future tasks.
