# VISUAL_FACTORY_MINIMUM_V0_1

## Почему нужен Step 7
Step 7 вводит контролируемую visual factory, чтобы визуальная эволюция Sanctum была проверяемой и не ломала runtime.

## Two-Phase Flow
- Phase 1 (этот шаг): структура, манифесты, baseline rules, checker.
- Phase 2 (позже): intake screenshot evidence, interpretation cards, Owner confirmation, controlled promotion в канон.

## Folder Map
- `ASSETS/` — evidence/reference зона.
- `SANCTUM/DESIGN_SYSTEM/` — starter tokens/rules/budgets.
- `SANCTUM/UI_LAB/` — isolated experiment zone.

## Owner Screenshot Drop
Owner кладёт raw/annotated screenshots в `ASSETS/INBOX_OWNER_VISUAL_SORTING/`.
Servitor обрабатывает их как proposal, не как canon.

## Canon Gate
Канонизация разрешена только после явного Owner confirmation.

## UI Change Gate
Runtime-визуальные изменения допускаются только после UI_LAB evidence и review.

## No Fake Green
Запрещены заявления о завершённой visual registration без подтверждённых evidence и approvals.
