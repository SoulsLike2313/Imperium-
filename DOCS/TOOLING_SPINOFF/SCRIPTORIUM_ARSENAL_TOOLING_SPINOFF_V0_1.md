# SCRIPTORIUM + ARSENAL Tooling Spin-Off v0.1

Статус: active support track v0.1

## Смысл трека

Этот трек регистрирует поддержку инструментов и скриптов как слой возможностей IMPERIUM.
Это **support track**, а не новый canon organ.

## Позиционирование

- `SCRIPTORIUM` — слой реестра/индекса внутренних скриптов IMPERIUM.
- `ARSENAL` — слой реестра внешних tools/capabilities.
- Трек ускоряет навигацию, усиливает доказуемость и повышает безопасность планирования.

## Кто использует

- Servitors и Logos-Prime для выбора допустимых инструментов и скриптов по scope.
- Kiro/другие advisory reviewers для технического аудита capability-gap.
- Будущие органы для task/stage/run planning и проверок.

## Границы v0.1

- Этот трек не создает новый канонический орган автоматически.
- Registered tool/script != installed/approved tool/script.
- Recommended != approved.
- Любая установка требует явного Owner decision.

## Связь с TASK/STAGE/RUN

`TASK`, `STAGE`, `RUN` могут ссылаться на эти реестры как на support capability metadata:
- allowed scripts,
- required capabilities,
- owner-gated tools,
- forbidden tools.

## Governance

- Administratum может использовать данные для маршрутизации и handoff-пакетов.
- Mechanicus отвечает за техническую пригодность script/tool spine.
- Inquisition использует слой для risk/safety проверок.
- Sanctum может строить кнопки только поверх безопасно зарегистрированных команд.

## No Fake Green

- Нельзя заявлять readiness по инструментам без фактической проверки.
- Нельзя маркировать installed, если это не подтверждено contour-local evidence.
- Нельзя обходить Owner gate для install/commit/push/sync.
