# IMPERIUM — OWNER DOCTRINE

## Назначение

Эта папка фиксирует Owner-level доктрину 10 органов IMPERIUM и advisory-усиление этой доктрины.

Это не код, не roadmap и не исполняемый orchestrator. Это каноническая смысловая основа, на базе которой позже будут строиться organ scaffolds, registry sync, ports, schemas, scripts, receipts и Sanctum HUD.

## Статус файлов

- `OWNER_ORGAN_DOCTRINE_V0_1.md` — канонический источник Owner-смысла и порядка органов.
- `OWNER_ORGAN_DOCTRINE_REGISTRATION_PLAN_V0_1.md` — pre-commit / registration plan: как использовать доктрину и не дать внешним аудиторам менять порядок.
- `KIRO_OWNER_ORGAN_DOCTRINE_ADVISORY_20260512.md` — read-only advisory pass от Kiro. Полезные инженерные советы, но не канон.

## Каноническое правило

Owner doctrine имеет приоритет над Kiro / Speculum / любым агентом.

Kiro advisory может:
- усиливать;
- предлагать файлы, порты, схемы, receipts, HUD;
- указывать риски и блокеры.

Kiro advisory не может:
- менять порядок органов;
- переименовывать органы;
- подменять Throne обычной кнопкой accept;
- переносить Strategium в начало цепочки;
- раскрывать приватную архитектуру как публичное портфолио.
