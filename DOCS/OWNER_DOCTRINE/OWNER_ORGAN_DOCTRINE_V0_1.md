## 1. Смысловая цепочка задачи

```text
TASK_ID enters system
→ Doctrinarium checks if the Imperium is alive and lawful
→ Administratum gives task address, workspace, memory, registration
→ Officio Agentis gives executor role, limits, allowed behavior
→ Astronomicon gives General Task / Task / Stage Map / pass criteria
→ Mechanicus gives tools, scripts, command routes, machinery
→ Servitor executes stage
→ Inquisition audits diff, artifacts, meaning, cleanliness
→ Administratum records stage completion
→ Astronomicon advances stage
→ repeat until task completion
→ Throne / Custodes protect accepted canon
→ Strategium researches improvements asynchronously by Owner command
→ Schola Imperialis creates Owner learning path by Owner command
```

## 2. Owner decisions — canon contour and advisory triggers

### Throne contour

Throne — это не просто папка, не просто branch и не просто script.  
Throne в Owner-доктрине — это **весь ноут как защищённый канонический контур**, который питает ядро IMPERIUM своей железной независимостью.

Техническая реализация пока не финализирована, но доктринально:

```text
THRONE = protected laptop-based canonical kernel contour
```

На v0.1:

```text
Throne is OWNER-controlled.
Throne is not auto-written.
Throne accepts only verified, clean, Owner-approved states.
```

### Custodes backup policy

Clean backup держится на внешнем диске.

Custodes отвечает за то, чтобы:

```text
- проверить, что состояние принято;
- проверить, что оно не грязное;
- создать clean backup на external drive;
- записать backup receipt;
- не пустить непроверенное в canonical backup.
```

На v0.1:

```text
CUSTODES_BACKUP_TARGET = external drive
CUSTODES_TRIGGER = Owner command only
CUSTODES_BACKUP_TYPE = clean accepted state snapshot
```

### Strategium trigger

Strategium запускается только по Owner-команде.

```text
STRATEGIUM_TRIGGER = OWNER_COMMAND_ONLY
STRATEGIUM_MODE = low-resource advisory research
STRATEGIUM_OUTPUT = research findings, links, improvement ideas
STRATEGIUM_AUTHORITY = advisory only, no automatic code/action
```

### Schola Imperialis trigger

Schola Imperialis запускается только по Owner-команде.

```text
SCHOLA_TRIGGER = OWNER_COMMAND_ONLY
SCHOLA_MODE = Owner learning advisory
SCHOLA_OUTPUT = learning notes, topics, reading path, skill gaps
SCHOLA_AUTHORITY = advisory only
```
