# ADMINISTRATUM STAGE ROUTING PROTOCOL V1

Inputs:
- structured owner prompt
- allowed contours
- constraints and blocked actions

Outputs:
- stage decomposition
- contour assignment
- dependency map
- stage pass criteria
- required tool IDs
- stop/repair policy hooks

Constraints:
- no fallback routing
- no unknown contour assignment
- all stages bound to TASK_ID/STAGE_ID/RUN_ID model
