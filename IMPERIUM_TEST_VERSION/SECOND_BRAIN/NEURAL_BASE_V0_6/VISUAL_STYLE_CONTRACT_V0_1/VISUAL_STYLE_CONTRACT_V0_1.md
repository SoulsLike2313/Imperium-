# VISUAL_STYLE_CONTRACT_V0_1

## Scope
Applies only to `IMPERIUM_TEST_VERSION/SECOND_BRAIN/NEURAL_BASE_V0_6`.

## Core Rule
`TRUTH > BEAUTY`.

If visual polish conflicts with truthful backend representation, truth wins.

## Style Formula
Dark purple nebula metallic + sci-fi + holographic semantic + Jarvis-like animation + 60fps-feeling motion + rich minimalism + high animation quality + 1080-1440p minimal texture.

## Visual Identity Requirements
1. The neural map must read as a coherent brain-region field, not empty space.
2. The core brain must be visible immediately on first load.
3. All 12 zones must be visible without hover.
4. Zone labels must be readable at 1080p.
5. Strands must read as neural pathways and remain visible but not noisy.
6. Right operator panel must feel active, metallic, readable, and actionable.
7. Task Intake Corridor entry must remain obvious.
8. Visual atmosphere must not hide controls or truth counters.

## Truth Preservation Rules
1. No fake counters.
2. No fake statuses.
3. No fake health state.
4. No fake task/package/comment/link/receipt event.
5. No fake external agent execution implication.
6. No hardcoded non-decorative values that override backend truth.
7. Non-decorative UI values must remain backend-bound or honestly marked unavailable.
8. Partial/blocked/missing states must remain honest and visible.
9. Honesty badges must remain explicit:
   - `PROTOTYPE_INTERACTIVE`
   - `RULE_BASED_ONLY`
   - `NO_LOCAL_LLM`
   - `NO_AGENT_API`
   - `NOT_RELEASE_READY`

## Brain Zone Requirements
1. Central neural core must be bright enough to anchor visual attention.
2. Zone nodes must look like neural regions with visible boundaries.
3. Hover must reveal region identity clearly.
4. Click must visually connect selected region to operator panel context.
5. Core + zone composition must stay readable at 1920x1080 and 2560x1440.

## Operator Panel Requirements
1. Typography and spacing must prioritize readability over compact density.
2. Active tab must be obvious.
3. Chips/cards must have strong contrast and hierarchy.
4. Task list must remain readable at a glance.
5. Corridor controls must remain prominent and operational.

## Animation Requirements
1. Motions should feel smooth and stable (60fps-feeling where possible).
2. Allowed motion: zone pulse, strand flow, hover elevation, panel transitions, subtle ambient motion.
3. Avoid chaotic effects that mask data.
4. No animation may imply state changes that did not occur in backend.
5. Reduced-motion fallback should be available when possible.

## Acceptance Gates
1. 12 zones visible without hover.
2. Labels readable.
3. Core visible.
4. Right panel readable and populated.
5. Truth bar readable.
6. No console errors during normal flow.
7. Task Intake Corridor full flow still works (register/comment/link/launch/handoff).
8. Receipts still created by real actions.
9. No truth regression against baseline.

## Non-Goals for this pass
1. No new major feature workflows.
2. No backend architecture rewrite.
3. No production-ready claim.
4. No commit/push.
