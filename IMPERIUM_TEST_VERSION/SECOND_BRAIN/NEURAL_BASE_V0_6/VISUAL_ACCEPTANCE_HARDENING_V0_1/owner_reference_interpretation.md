# owner_reference_interpretation

## 1) Target patterns (from Owner references)
1. Centralized neural composition: strong center anchor with radial/branching energy lines.
2. Brain-field identity: the main canvas should read as one coherent sci-fi brain region, not isolated dots.
3. Purple-cyan holographic palette: deep dark background, electric cyan and violet with warm amber accents.
4. Layered depth: atmospheric nebula + orbital rings + internal neural fiber texture.
5. Zone identity blocks: each capability should feel like a region/node with integrated label, not tiny detached captions.
6. Premium operator surface: right panel must be a strong cockpit block with clear hierarchy and large readable text.
7. Controlled motion: smooth pulses/flows and panel transitions, no chaotic visual noise.
8. Last mock screenshot is the primary target direction: organized circular brain system with readable labeled capability capsules around core.

## 2) Anti-patterns (to avoid)
1. Empty dark field with weak structure.
2. Generic node-map look with only small glow changes.
3. Microscopic chips/labels.
4. Weak right panel with low hierarchy.
5. Random decorative effects that hide data.
6. Any fake status/count/health implication.

## 3) Must-fix areas in current V0.6
1. Strengthen the brain-field silhouette and internal neural texture.
2. Make zone blocks read as integrated brain regions with better text readability.
3. Increase right panel readability and visual authority.
4. Improve truth/top bar legibility at 1080p.
5. Preserve responsive behavior while increasing visual density quality.

## 4) Must-preserve areas
1. Backend-truth bindings for counters/status/health/snapshot identity.
2. 12-zone model and interactivity.
3. Task Intake Corridor full functional flow (register/comment/link/launch/handoff).
4. Honesty badges: PROTOTYPE_INTERACTIVE / RULE_BASED_ONLY / NO_LOCAL_LLM / NO_AGENT_API / NOT_RELEASE_READY.
5. Real receipt generation and runtime file side effects from real interactions.

## 5) Truth risks to avoid
1. Hardcoding non-decorative counts or statuses into visual labels.
2. Showing WORKING-like visual cues for zones that are PARTIAL/BLOCKED/MISSING.
3. Animated cues that imply backend events that did not occur.
4. Breaking corridor interactions while changing panel/canvas structure.
5. Resize-induced JS/CSS desync where DOM still shows stale data.
