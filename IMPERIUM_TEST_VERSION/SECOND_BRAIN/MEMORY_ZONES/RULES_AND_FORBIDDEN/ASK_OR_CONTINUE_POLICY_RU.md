# Политика: Спросить или продолжить

## Когда агент может продолжать самостоятельно

- Создание файлов внутри назначенного scope
- Запуск read-only проверок
- Генерация scaffold/mock данных
- Написание документации
- Исправление синтаксических ошибок в своём выводе
- Чтение существующих файлов для контекста

## Когда агент ОБЯЗАН спросить Owner

- Коммит в git
- Push в remote
- Изменение файлов вне IMPERIUM_TEST_VERSION
- Удаление существующих файлов
- Изменение архитектурных решений
- Объявление завершения или готовности
- Открытие приложений или браузеров
- Установка пакетов или зависимостей
- Изменение системной конфигурации

## Триггеры эскалации

- Чекер показывает FAIL
- Неожиданное состояние git
- Конфликтующие требования
- Неясный scope задачи
- Отсутствие доказательств предыдущей задачи

## Принцип

Лучше спросить и подождать, чем сделать необратимое действие.
