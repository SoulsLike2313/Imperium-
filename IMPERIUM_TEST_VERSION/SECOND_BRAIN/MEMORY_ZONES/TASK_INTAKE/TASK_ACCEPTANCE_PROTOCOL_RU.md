# Протокол приёма задачи

## Шаги

### 1. Приём (ACCEPTED)
- Задача получена
- Записана в TASK_INTAKE_INDEX.json
- Статус: ACCEPTED

### 2. Понимание (UNDERSTOOD)
- Агент формулирует своё понимание задачи
- Определяет scope
- Определяет ограничения
- Статус: UNDERSTOOD

### 3. Контекст (QUESTIONS_PENDING или ROUTED)
- Агент ищет релевантные зоны памяти
- Если есть неясности — формулирует вопросы
- Если всё ясно — предлагает маршрут
- Статус: QUESTIONS_PENDING или ROUTED

### 4. Маршрутизация (ROUTED)
- Определён агент/контур для исполнения
- Определены инструменты
- Определены критерии завершения
- Статус: ROUTED

### 5. Исполнение (IN_PROGRESS)
- Задача передана в Execution Memory
- Фиксируются этапы, решения, блокеры
- Статус: IN_PROGRESS

### 6. Завершение (COMPLETED или BLOCKED)
- Результат зафиксирован
- Доказательства собраны
- Статус: COMPLETED или BLOCKED

## Правила

- Никогда не пропускать шаг "Понимание"
- Вопросы лучше задать сейчас, чем переделывать потом
- Блокер — это нормально, скрытый блокер — это FAIL
