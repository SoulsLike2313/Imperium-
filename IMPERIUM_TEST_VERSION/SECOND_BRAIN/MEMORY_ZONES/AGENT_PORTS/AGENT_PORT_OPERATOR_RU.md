# Оператор: Agent Ports

## Статус: MOCK

## Что это

Зона для интеграции внешних, облачных, API, CLI и локальных агентов.
V0.2 содержит 3 примера портов Kiro (A, B, C) и схемы протоколов.

## Как будет работать (будущее)

1. Задача поступает в Task Intake
2. Система определяет какой агент/порт подходит
3. Формируется контракт (agent_task_contract)
4. Агент получает задачу через свой порт
5. Агент работает, отправляет статусы
6. При завершении — receipt + handoff если нужно
7. При конфликте — owner_decides

## Протоколы

| Протокол | Назначение |
|----------|-----------|
| task_assignment | Назначение задачи агенту |
| status_update | Обновление статуса |
| question | Вопрос к Owner или другому агенту |
| answer | Ответ на вопрос |
| receipt | Подтверждение завершения |
| handoff | Передача задачи другому агенту |
| conflict_report | Сообщение о конфликте |
| group_coordination | Координация группы агентов |

## Что НЕ реализовано

- Реальная маршрутизация задач
- Реальный обмен сообщениями между агентами
- Реальная координация группы
- Автоматическое разрешение конфликтов
- Очередь задач для агентов

## Масштабирование

Архитектура поддерживает сотни портов.
V0.2 показывает 3 как proof of concept.
