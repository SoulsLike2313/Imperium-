# STOP & ESCALATION RULES

## Когда агент ДОЛЖЕН остановиться

### Критические STOP условия

1. **Неясность цели** — если цель Owner непонятна
2. **Выход за scope** — если работа требует изменений вне разрешённых путей
3. **Forbidden action** — если требуется запрещённое действие
4. **Архитектурное решение** — если нужно менять архитектуру
5. **Безопасность** — если действие может навредить системе
6. **Данные** — если требуется доступ к приватным данным

### Мягкие STOP условия

1. **Неоднозначность** — несколько равноценных вариантов
2. **Риск** — действие с высоким риском
3. **Время** — задача занимает больше ожидаемого
4. **Зависимости** — нужны внешние ресурсы

## Формат STOP сообщения

```
[STOP]
TASK: <task_id>
REASON: <reason>
CONTEXT: <context>
OPTIONS:
- <option1>
- <option2>
RECOMMENDED: <option>
OWNER_ACTION_REQUIRED: YES
```

## Эскалация

### Уровни эскалации

1. **INFO** — информирование, работа продолжается
2. **WARNING** — предупреждение, работа продолжается с осторожностью
3. **BLOCKER** — блокер, работа остановлена до решения Owner
4. **CRITICAL** — критическая проблема, требуется немедленное внимание

### Формат эскалации

```
[ESCALATION]
LEVEL: <level>
TASK: <task_id>
ISSUE: <issue>
IMPACT: <impact>
URGENCY: <urgency>
OWNER_ACTION: <action>
```

## Правила

1. **Лучше остановиться, чем сломать** — при сомнениях STOP
2. **Честность** — не скрывать проблемы
3. **Варианты** — всегда предлагать варианты решения
4. **Рекомендация** — указывать рекомендуемый вариант
5. **Контекст** — давать достаточно контекста для решения
